# STL数据结构相关：

## std::array

底层实现就是对T[N]的包装，只有一个成员变量T[N]，因此sizeof(std::array<T, N>) == sizeof(T[N])

没有虚函数，没有动态内存，因此是零开销抽象，比原生数组好在，有大小，支持STL算法，有类型系统，编译期可以防止错误，边界检查等。且没有性能损耗，因为底层一样，就像是ranges的views

在C++里，除了极其底层可能使用数组之外，其他全都用std::array替代调数组。

## std::vector

是一块连续内存上的可自动扩容的动态数组，支持O(1)的随机访问和尾部插入。

内部基本靠三个指针，begin_指向第一个元素，end_最后一个元素，cap_指向已分配空间的末尾。空间是[begin_,cap_)
| 已构造对象 | 已构造对象 | 已构造对象 | 未使用但已分配 | 未使用 |
^            ^            ^             ^              ^
begin_       end_         size()         capacity()     cap_

容量不够时，需要分配一块更大的连续内存(标准不规定扩容比例，但GCC一般是接近2，MSVC是1.5)，并把所有元素move/copy到新内存，再销毁旧元素和内存

reserve(n)
如果capacity < n，则内存扩容到n，不构造新元素
如果capacity >= n，则什么都不做

resize(n)
n > size()，则扩大size()到n，构造新元素
n < size()，则size缩小，析构多余元素，capacity不变

clear()：只析构元素，不释放内存，一般是O(n)，但如果存的全是基本类型无析构函数的，则可以是O(1)

释放内存：
shrink_to_fit是非强制的，
因此真正释放内存：swap一个空vector。且注意swap参数不能是右值，因此不能临时构建个空的再swap，可以构建个空的来swap后不再使用

vector中间插入慢原因：为了保证连续内存，需要将插入点的元素整体后移

vector内的item永远都在堆上

## deque的底层数据结构是什么？它的内部是怎么实现的？

靠中央控制表map结构来访问多段连续内存块buffer，还有起始和结束迭代器

## map和unordered_map有什么区别？分别在什么场景下使用？

map：靠红黑树，key是有序的

unordered_map：靠哈希表(bucket + 链表/红黑树)，key无序

大部分时候都是unordered_map更快更好，但如果要有序，有范围等就只能map。
但并不总是unordered_map更快，如果hash冲突严重，或者数据量小都可能更慢

## set和unordered_set

数据结构层面基本与map系列相同，但语义和接口层面不同

## list

一般不使用。用在已知迭代器位置的频繁插入/删除，把节点从一个list挪到另一个list等等，

std::find只要是可迭代的就可以传入，但传入list时是O(n)，不适合大量查找和访问

## std::ranges::alg 是处理range的，比如sort，find等，不能处理视图！ranges不是视图，是一个概念，是实际存在内存里的

## std::ranges::sort projection
std::ranges::sort(ranges, comparator, projection) 看代码里comparator 默认是 ranges::less， projection默认是identity
实际比较是comparator(projection(ranges(item)))

projection更推荐返回值语义，所以使用std::tuple{},comparator可以使用引用，因此std::tie{}

# STL：

## 排序：

| 算法           | 用途             |
| ------------ | -------------- |
| sort         | 全排序            |
| stable_sort  | 稳定排序           |
| partial_sort | TopK（有序）       |
| nth_element  | 第 K / TopK（无序） |

一般用于topk问题：K从0开始
结果有序就partial_sort：
partial_sort(v.begin(), v.begin() + k, v.end());
// v[0..k-1] 是最小的 k 个（有序）

无序就nth_element：
nth_element(v.begin(), v.begin() + k, v.end());
// v[0..k-1] 是最小的 k 个（无序）

第K大就：
nth_element(v.begin(), v.begin() + k, v.end(), greater<>());
auto kth = v[k];

频率topK就先map统计频率，再自定义比较器：
unordered_map<int,int> cnt;
for (auto x : v) cnt[x]++;
vector<pair<int,int>> a(cnt.begin(), cnt.end());
partial_sort(a.begin(), a.begin() + k, a.end(),
    [](auto& x, auto& y){ return x.second > y.second; });

top K 实时最值

最大堆：
priority_queue<int> max_heap;
// top出来的永远是最大元素
max_heap.push(3);
max_heap.push(1);
max_heap.push(5);
max_heap.top(); // 5


最小堆：
priority_queue<int, vector<int>, greater<>> min_heap;

维护topK
priority_queue<int, vector<int>, greater<>> pq;
for (int x : v) {
    pq.push(x);
    if (pq.size() > k) pq.pop();
}

## 遍历 / 计数 / 判断

## 删除 / 移动

remove：将不等于给定值元素往前移，相当于将等于特定值的元素移动到末尾
remove_if：将不满足条件的元素往前移，相当于将满足条件的元素移动到末尾
unique：相邻元素相同的只保留一个，因此一般需要先排序再unique可以去重
这些是算法，算法不能改变容器大小，只能遍历，比较，重排等，只是移动，不删除元素。
只有容器操作才能处理，需要配合erase

去重：
sort(v.begin(), v.end());
v.erase(unique(v.begin(), v.end()), v.end());

按条件删除：
v.erase(remove_if(v.begin(), v.end(),
    [](int x){ return x < 0; }), v.end());

# 简单场景，不关联具体STL算法

## 区间合并问题：
vector<pair<int,int>> intervals;
sort(intervals.begin(), intervals.end());
vector<pair<int,int>> res;

for (auto& [l, r] : intervals) {
    if (res.empty() || res.back().second < l)
        res.push_back({l, r});
    else
        res.back().second = max(res.back().second, r);
}

## 滑动窗口：dequeue当作双端队列
deque<int> dq;
for (int i = 0; i < n; ++i) {
    while (!dq.empty() && a[dq.back()] <= a[i]) dq.pop_back();
    dq.push_back(i);
    if (dq.front() <= i - k) dq.pop_front();
}

## 单调栈
stack<int> st; // 存放的是下标
for (int i = 0; i < n; ++i) {
    while (!st.empty() && a[st.top()] < a[i]) st.pop();
    st.push(i);
}
