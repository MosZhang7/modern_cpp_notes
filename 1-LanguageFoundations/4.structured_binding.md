# structured binding C++17

有点类似于语法糖，auto返回内部类型，pair或者tuple，通过.get获取值

结构化绑定只能绑定三种类型
1. 数组:
``` C++
int a[3] = {1,2,3};
auto [x, y, z] = a;
```
2. tuple-like: std::pair, std::tuple, std::array，以及ranges::views::zip后的内容
3. Aggregate聚合类型

聚合类型在 C++17 的定义：
- 没有任何自定义构造函数
- 没有 private/protected 的非静态成员（成员都得 public）
- 没有虚函数
- 没有虚基类
例如
``` C++
struct A {
    int x;
    double y;
};           // ✔ 是 aggregate → 可以结构化绑定
```

最常用于for循环内解构auto对象的item，例如vector，map等，例如：
``` C++
std::map<std::string, int> m = {{"A", 1}, {"B", 2}, {"C", 3}};

for (auto&& [key, value] : m) {
    value += 10;
}
```

# std::tie

C++11和std::tuple一起出现，主要用于解tuple的包，用于生成一个tuple里数据的引用视图，不拷贝不拥有数据。

在C++17有了结构化绑定之后就不再用于解包

C++20后则主要用于多字段以不同key进行比较或排序。因为只是引用tuple-like里的元素，因此开销很小，可以合包作为视图，且tuple天然严格弱序

主要用于comparator，而不是projection，projection更推荐值语义
