# 模板参数的匹配

模板推导规则里T为非引用类型，万能转发的情况下才可能会推导出T&

先匹配形参，再反向推导出模板参数T。


例如：
``` C++
template<class T>
void f(T x);

int a = 10;
f(a); // a按值传递，因此形参x的类型为int，所以T为int，x的实际类型为int

// ---------------

template<class T>
void f(T& x);

int a = 10;
f(a); // a按值传递，能绑定到值上的只有int和int&，而形参x由于有引用，因此x是int&类型，所以T为int
const int ca = 10;
f(ca); // ca按值传递，能绑定到ca上的只有const int和const int &，因此T& x只能是const int &，所以T为const int


// ---------------

template<class T>
void f(T&& x);

int a = 10;
f(a); // a为左值传入，能绑定到a的只有int & x，因此x是int&类型，所以T为int&形成引用坍缩
f(10); // 右值传入，能绑定到右值的只有int &&和const int&，因此x为int&&，所以T是int

// ---------------

template<class F, class... Args>
void submit(F&& f, Args&&... args);

auto lam = []{ return 1; };
submit(lam); // 传lambda左值，F被推导为lam_type &

submit([]{ return 1; }); //传lambda右值，F被推导为 lam_type

// 这时候如果


```



# 外部模板

由于用到的地方就会进行实例化，就会导致重复实例化，因此在头文件中可以声明成外部模板

``` C++
template class std::vector<bool>;          // 强行实例化
extern template class std::vector<double>; // 不在该当前编译文件中实例化模板
```

# 变长参数模板

现代C++里，...在C++里代表两个概念：**参数包声明**和**参数包展开**，且只能出现在模板相关的使用里，因为早期C风格函数的变长参数没有类型，不安全。

1. 参数包声明：不做展开，不拆包，更不会运算
template <typename... Args>
void f(Args... args);
表示Args是一个参数包类型，就像int &r 里的&一样表示是个引用

2. 参数包展开：拆包只是展开，不做运算，折叠表达式才做运算。
模板函数包展开后，相当于一元调用多个参数的函数，只调用一次普通函数，并不是分多次调用单个参数的函数。
``` C++
template <typename... Args> // 声明为参数包
void foo(Args... args) { // 声明为值包
    bar(args...);   // 参数展开
}
// 展开后等价于
bar(arg1, arg2, arg3);
```

3. init-capture
...as = std::forward<Args>(args)
在lambda表达式的捕获列表里，重复的是as = std::forward<Args>(args)，变为as[0] = xxx, as[1] = xxx等
as是参数包，且lambda表达式内部可以使用as...拆包

## 折叠表达式

| 名称                           | 标准语法                    | 适用情况      | 展开规则                             |
| ---------------------------- | ----------------------- | --------- | -------------------------------- |
| **Unary Right Fold**  | `(pack op ...)`         | 右结合       | `((a op b) op c) op d`           |
| **Unary Left Fold**   | `(... op pack)`         | 左结合       | `a op (b op (c op d))`           |
| **Binary Right Fold** | `(pack op ... op init)` | 指定初始值，右折叠 | `(((a op b) op c) op d) op init` |
| **Binary Left Fold**  | `(init op ... op pack)` | 指定初始值，左折叠 | `init op (a op (b op (c op d)))` |

## 模板全特化/偏特化

现代C++基本不再使用，通过 if constexpr 和 concept 代替

全特化示例：对模板的所有模板参数给出确定类型。
``` C++
template <typename T>
struct Printer {
    static void print() { std::cout << "generic\n"; }
};

// 全特化
template <>
struct Printer<int> {
    static void print() { std::cout << "int\n"; }
};

```

偏特化示例：只对模板参数的一部分进行约束，或者通过类型模式进行匹配。

**只能用于类模板，不能用于函数，函数偏特化可以通过重载代替掉**
``` C++
template <typename T>
struct TypeInfo {
    static constexpr const char* name = "general";
};

// 偏特化：所有指针类型
template <typename T>
struct TypeInfo<T*> {
    static constexpr const char* name = "pointer";
};
```
