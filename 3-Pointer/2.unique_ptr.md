# unique_ptr

本质是个小对象，里面只保存原始的裸指针 T* 和一个deleter D，析构时直接if (ptr != nullptr) deleter(ptr);

由于独占所有权，因此拷贝构造/拷贝赋值 = delete

deleter决定了不光可以管理new/delete，还可以管理资源文件如FILE* 和socket_fd等

deleter不能传lambda，因为函数签名里，第二个模板参数是类型，lambda表达式是匿名类型，无法获取。因此一般用callable类，或者通过decltype来声明lambda的类型

常用方法：
- release():交出指针，自己变空，不析构，不再管理这个指针
- reset(p):析构当前对象，再接管 p，也可以不传参，直接析构当前对象
- swap:交换两个的对象指针和deleter指针
