# weak_ptr

与shared_ptr类似，也有control_block*，有没有T*看实现，标准不规定，一般没有。

weak_ptr不能凭空存在，因为要指向控制块，因此构造里要么来自shared_ptr，要么来自另一个weak_ptr

一般通过if (auto sp = wp.lock())获得一个强引用 sp 再进行操作

常用方法：
- lock:尝试升级为shared_ptr，如果对象已析构，返回空shared_ptr。
- reset():不再观察
- swap:交换内部control_block的绑定
- use_count:跟shared_ptr一样返回strong_count

expired：基本没用，除了调试可能有用

lock时底层是线程安全的，weak_count和strong_count都是原子变量
if (auto sp2 = wp.lock()) {
    // 安全使用
}
