#  std::async / std::packaged_task

async和packaged_task都是对promise的封装，可能会产生出future

# std::async

底层实现类似于对 promise / future 和 thread 的封装，返回一个future

async 启动的“执行体”，其生命周期必须与返回的 future 绑定，因此析构future可能阻塞，因为要等async的callable跑完，类似于join

因此如果不接收async直接启动，会导致退化为串行，因为一启动就原地析构掉了返回值future，会需要等待callable运行完毕

## async的启动参数：

std::launch::async：立刻创建新线程开始执行异步任务
std::launch::deferred：等到future.wait()或者get()才会开始运行。此时wait_for()/wait_until()返回状态为deferred

默认为std::launch::async|std::launch::deferred，根据标准库的实现决定，因此应显示指出

auto f = std::async(std::launch::async, func);

# std::packaged_task

构建时，创建shared state，再通过set_future生成与promise一样规则的future。调用packaged_task时，自动set_xxx

调用时，原理类似于：
``` C++
try {
    if constexpr (std::is_void_v<R>) {
        func();
        state->set_value();
    } else {
        state->set_value(func());
    }
} catch (...) {
    state->set_exception(std::current_exception());
}
```

简单示例：
``` C++
std::packaged_task<int()> task([] {
    return 42;
});

auto fut = task.get_future();

task();            // 执行
int x = fut.get(); // 42

// 或者与std::thread结合，因此很适合线程池
std::thread t(std::move(task));
t.detach(); // 或 join

int x = fut.get();
```

一个task只能被执行一次，无法两次运行。因此只能被move不能被copy，是一次性计算任务

task没被执行就被析构，相当于promise还没set_xxx就被析构，会将异常写入shared state并将状态ready，唤醒等待线程
