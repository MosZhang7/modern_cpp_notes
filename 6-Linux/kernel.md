# 内核实时

jitter：抖动
软实时：有时间要求，但偶尔超时不算错误，例如视频播放
硬实时：任何一次超时都被视为错误，造成严重后果
deadline：通常等于周期，表示这一周期内必须完成的最晚时间点

可能带来延迟的因素：
1. 代码逻辑，例如加锁，等待等
2. 调度策略，或运行时被抢占，page fault/内存分配
3. CPU频率变化，cache miss等

# Linux内核调度

调度策略是绑定到线程上，不是进程不是内核，而调度策略的换上CPU都是针对单个计算单元，有多个核心时可以把不同任务绑定到不同CPU上进行独占

Linux默认CFS，实时系统一般SCHED_RR，极端情况用SCHED_FIFO

## SCHED_RR
同优先级线程之间，会轮转时间片

## SCHED_FIFO
优先级高的永远先跑，不主动让出CPU，就不会被换下

# CPU亲和性

即便什么都没写，CPU也会被中断打断。中断的优先级极高，会打断FIFO的线程，为了防止这种情况带来的jitter，就是不让中断跑在实时CPU上
